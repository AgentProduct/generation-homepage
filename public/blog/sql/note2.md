## 关于缓存

### Cache Aside 缓存策略

把高频读、低频改的数据，放到更快的存储里，减轻数据库压力。

读多写少场景：用户信息、商品详情。

- 读：先查缓存 → 没有再查 DB → 查到后回写到缓存
- 写：更新 DB → 删除缓存（不是更新）

**缓存三大问题**

- 缓存穿透：查根本不存在的数据，缓存永远不命中，全打 DB
  - 解决：缓存空值，布隆过滤器
- 缓存雪崩：大量 key 同一时间过期 或 Redis 宕机
  - 解决：设置过期时间，集群、多级缓存，限流、降级
- 缓存击穿：一个热点 key 过期，瞬间大量请求打 DB
  - 解决：互斥锁，热点 key 永不过期

### 缓存失效

缓存失效就是指将缓存中的数据删除（或标记为过期），使得后续对该数据的访问无法命中缓存，必须重新从数据库加载。

在 Cache Aside 模式中，“写操作时删除缓存”这个动作，就是主动让缓存失效，确保后续读取的是最新数据，保证数据的一致性。

### 慢查询

慢查询就是执行时间过长、拖慢系统的 SQL 语句，需要优化。

常见原因：

- 没加索引，全表扫描
- 索引失效（函数运算、隐式类型转换）
- 一次查的数据太多：SELECT \*、LIMIT 太大
- 关联表太多、JOIN 没优化
- 数据量巨大，库表没分库分表
- 数据库压力大、锁等待、排队执行
